# CIM-Design: 纯软件模拟存算一体拓扑流形架构

在冯·诺依曼约束下逼近类脑流形计算的三层软件模拟架构。

## 核心思想

存算分离是 LLM 三大瓶颈（内存墙、拓扑固化、离散生成）的物理根源。本架构通过纯软件模拟，在标准 GPU 上实现类脑流形计算的核心特性——信息沿拓扑结构自发弛豫至低熵稳态而"涌现"，而非被逐层"计算"出来。

## 三层架构

| 层级 | 模块 | 解决的瓶颈 | 核心机制 |
|------|------|-----------|---------|
| L1 微观 | SRAM 融合引擎 | 内存墙 | 算子融合 + 分块计算 + 异步预取 |
| L2 中观 | 动态可塑性路由 | 拓扑固化 | MoE + TTT 在线学习 + EWC 制动 |
| L3 宏观 | 连续时间弛豫引擎 | 离散生成 | DEQ 不动点迭代 + 谱归一化 + 断路器 |

辅以**小世界拓扑网络**（全局流形基底）、**相变采样器**（连续→离散 Gumbel-SDE）、**跨尺度反馈**（L3→L1 闭环）。

## 理论基础

基于"三元生成内核 + 惯性制动"的涌现动力学框架：

1. **破缺**（对称性打破）→ 注入差异种子
2. **简单规则**（局域同质交互）→ 迭代放大差异
3. **时空时序**（粗粒化与尺度跃迁）→ 微观到宏观的层级跃迁
4. **惯性制动**（约束阻尼）→ 维持结构稳定

## 关键结果

- 存算一体模块额外开销：**52.4ms**（基准推理的 0.25%）
- SRAM 命中率：**100%**
- L1 融合引擎运算效率：**可忽略水平**

## 根本局限

软件无法消除物理数据搬运。真正的存算一体能效优势需要硬件级实现（忆阻器等）。本架构的价值在于：在现有硬件约束下，逼近类脑流形计算的**算法收益**。
