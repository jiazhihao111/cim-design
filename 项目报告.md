# 存算一体项目报告

## 项目定位

纯软件模拟存算一体拓扑流形架构，在标准 GPU（冯·诺依曼架构）上通过三层技术栈逼近类脑流形计算的算法收益。所有"存算一体"逻辑是**纯软件模拟层**，数据仍需在 HBM 和计算单元之间物理搬运——这是物理约束，软件无法绕过。

## 物理限制说明

以上修复让代码**逻辑正确**：L1 真正处理输入、拓扑网络真正影响计算、L3→L1 闭环打通、性能测量反映真实时间。

但**物理限制**仍然存在：所有修改依然是 PyTorch 层面的模拟。真正的"存算一体"优势需要：
- CUDA C++ kernel 融合（消除中间结果写回 HBM）
- FPGA/ASIC 的原位计算（物理上消除数据搬运）
- 模型架构级稀疏注意力（直接减少 FLOPs）
